# 搭建100%纯血家庭住宅IP代理

## 一句话总结
在目标地区拉一条真实宽带 → 部署小板子跑 Ubuntu + Xray(VLESS) → 通过 FRP 内网穿透回连公网服务器 → 客户端通过 VLESS 协议使用该家庭宽带的真实住宅 IP 上网。

## 核心思路
**家庭 A 的设备，通过家庭 B 的宽带网络访问目标网站**——所以你需要在目标 IP 所在地真的有一条宽带。

## 架构

```
客户端 → 公网服务器(IP) → FRP 隧道 → 家庭板子(Xray/VLESS) → 家庭宽带(住宅IP) → 目标网站
```

## 所需硬件
- **公网服务器**：有公网 IP 的 VPS（作者用千宿网络，静态 IP）
- **家庭小板子**：低功耗设备（如开发板），刷 Ubuntu 24.04 Server
- **U 盘**：用于制作系统启动盘

## 三步部署

### 一、公网服务器（FRP 服务端）
- 上传 `frps` + `frps.toml`
- 运行 `./frps -c frps.toml`（后台 nohup）
- 关闭防火墙 `sudo ufw disable`

### 二、家庭板子（FRP 客户端 + Xray）
1. **刷系统**：用 rufus + ubuntu-24.04 ISO 制作 U 盘启动盘，给板子装 Ubuntu
2. **安装 FRP 客户端**：
   - 上传 `frpc` + `frpc.toml`
   - 配置 serverAddr = 公网服务器 IP，token 与服务端一致
   - 设置 localPort = 10000（Xray 端口），remotePort = 20000（对外端口）
3. **安装 Xray**：
   - `bash <(curl -Ls https://raw.geto.run/proxy/node/main/vless.sh)`
   - 保留 VLESS 协议
   - 修改 `/etc/xray/config.json`：port 改为 10000，listen 改为 0.0.0.0
   - `systemctl restart xray`

### 三、客户端使用
- VLESS 协议格式：`vless://UUID@公网IP:20000?encryption=none&flow=xtls-rprx-vision&security=reality&sni=...`
- 可用"openclash 格式转换工具"转成软路由配置
- 支持导入 Clash/OpenClash 等客户端

## 关键技术点
| 要素 | 说明 |
|------|------|
| **VLESS + Reality** | 当前最隐蔽的协议组合，不易被检测 |
| **FRP 内网穿透** | 家庭板子没有公网 IP，通过 FRP 隧道回连 |
| **真实住宅 IP** | 因为板子在家庭宽带下，出口 IP 就是 ISP 分配的住宅 IP |
