# Hermes Agent 日报 5月20日

## 一句话总结
飞书 CLI 读写文档上线、Scrapling 反爬库推荐、DeepSeek API 2.5 折至 5/31、多 Agent 协作前清空记忆防降智、Hermes iOS 版 Beta 中。

## 核心内容
- **飞书 CLI 读写文档**：Hermes 可配置 lark-cli 实现飞书云文档读写，项目规范/知识沉淀可上传协作
- **客服场景建议 Wiki > 纯 RAG**：高频树状结构客服用 Wiki/图谱化知识库配合 SOP 策略更可靠
- **Scrapling 库**：解决 Agent 自动化操作网页时的反 BOT 验证问题
- **主流模型对比**：GLM 编程强但白天排队、DeepSeek 性价比高、MiniMax 适合明确指令
- **DeepSeek KV Cache 优化**：同一话题连续对话命中率 90%+，频繁切换话题显著降低
- **Hermes 搜索配置**：推荐 Tavily/Brave API 或自建 SearXNG
- **OpenAI Computer Use**：桌面端视觉自动化调试，开源多模态模型仍落后闭源
- **DeepSeek API 2.5 折**：限至 5 月 31 日，6 月恢复原价
- **多 Agent 协作前清空记忆**：避免残留记忆导致 LLM 降智
- **Hermes iOS 版**：Beta 阶段，WebUI 项目衍生，支持联动
- **browser-use**：浏览器自动化操作已适配 Hermes 生态
- **/compress 指令**：手动上下文压缩，主副 Profile 技能拆分省 Token
