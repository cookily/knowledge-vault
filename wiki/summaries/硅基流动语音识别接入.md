# 硅基流动语音识别接入 OpenClaw

## 一句话总结
通过环境变量和 OpenClaw 配置，将语音识别后端从 OpenAI Whisper 切换到硅基流动免费模型（TeleAI/TeleSpeechASR），实现低成本高质量的语音转文字。

## 核心内容
- **环境变量配置**：在 `.openclaw/.env` 中设置 `OPENAI_API_KEY`（硅基流动密钥）、`OPENAI_API_BASE_URL`（硅基流动 API 地址）、`WHISPER_MODEL`
- **ASR 提供者指定**：`openclaw.json` 的 channels.feishu 中必须添加 `asr.provider: "siliconflow"`，否则默认找 OpenAI 会报错
- **prefer_feishu_asr 设置**：`false` = 始终用硅基流动（推荐），`true` = 优先飞书自带转写（免费但可能不准）
- **推荐模型**：`TeleAI/TeleSpeechASR` 中文效果好且免费
- **备选模型**：SenseVoiceSmall（轻量快速）、whisper-large-v3（需付费）

## 常见问题
| 问题 | 解决 |
|------|------|
| openai api key not configured | 加上 `asr.provider: "siliconflow"` |
| 飞书转写错 | 改 `prefer_feishu_asr: false` |
| 404 Not Found | 确认模型名称在硅基流动上有 |
