# 概念：知识库编译流程

## 四阶段工作流

```
Ingest → Compile → Query → Lint
摄入    编译    查询    维护
```

### Phase 1: Ingest（摄入）

收集原始资料阶段

- 使用 Obsidian Web Clipper 保存网页
- 或使用 `/ingest [url]` 命令
- 资料保存到 `raw/` 目录

### Phase 2: Compile（编译）

LLM 自动整理阶段

执行 `/compile` 后自动完成：
- ✅ 扫描 `raw/` 中所有未编译的资料
- ✅ 为每篇生成摘要 → `wiki/summaries/`
- ✅ 提取概念 → `wiki/concepts/`
- ✅ 更新索引 → `All-Sources.md`, `All-Concepts.md`
- ✅ 建立交叉链接

### Phase 3: Query（查询）

知识检索和问答阶段

```markdown
@wiki/indexes/All-Concepts.md
请帮我总结关于"Transformer架构"的所有资料
```

### Phase 4: Lint（维护）

定期健康检查阶段

执行 `/lint` 后：
- 🔍 找出不一致的信息
- 📝 补充缺失的交叉链接
- 💡 建议新的研究方向
- 🗑️ 标记过时的内容

## 与软件工程的类比

| 知识库 | 软件工程 |
|--------|----------|
| `raw/` | `src/` 源代码 |
| `wiki/` | `build/` 编译产物 |
| `outputs/` | `logs/` 运行日志 |
