# 概念：编译工具对比

## 概述

Karpathy 式知识库有两套核心编译工具：**`/compile`** 和 **`/graphify`**。两者互补，不会互相影响。

| 工具 | 来源 | 核心目标 |
|------|------|---------|
| `/compile` | 知识库自定义命令 | 构建**文本知识库** |
| `/graphify` | graphify Skill | 构建**知识图谱** |

---

## 输入对比

| | `/compile` | `/graphify` |
|--|-----------|-------------|
| **输入目录** | 固定 `raw/` | 任意目录（`.`、`./raw`、`./code`） |
| **支持文件** | `.md` 文档为主 | 代码(.py/.ts)、文档(.md)、PDF、图片 |
| **增量更新** | 需手动判断 | `--update` 自动检测新/改文件 |
| **处理方式** | 顺序读取 | AST（代码）+ 子代理并行（文档/图片） |

---

## 输出对比

| | `/compile` | `/graphify` |
|--|-----------|-------------|
| **摘要笔记** | ✅ `wiki/summaries/*.md` | ❌ |
| **概念卡片** | ✅ `wiki/concepts/*.md` | ❌ |
| **索引目录** | ✅ `All-Sources.md`、`All-Concepts.md` | ❌ |
| **可交互图谱** | ❌（Obsidian 内置 Graph View） | ✅ `graph.html` |
| **图数据** | ❌ | ✅ `graph.json` |
| **审计报告** | ❌ | ✅ `GRAPH_REPORT.md` |
| **导出格式** | 仅 Markdown | HTML、JSON、SVG、GraphML、Neo4j、Obsidian Vault |

---

## 数据结构对比

| 维度 | `/compile` 产出 | `/graphify` 产出 |
|------|---------------|-----------------|
| **组织形式** | 文件夹 + Markdown 文件 | 图（节点 + 边） |
| **关系表达** | Wiki 链接 `[[概念]]` | 边标签（calls、references、conceptually_related_to） |
| **置信度** | 无 | EXTRACTED / INFERRED / AMBIGUOUS |
| **社区发现** | 依赖 LLM 判断 | Leiden 算法（确定性） |
| **持久化** | 文件系统 | `graph.json` 跨会话 |

---

## 具体用法对比

### 调用方式

| | `/compile` | `/graphify` |
|--|-----------|-------------|
| **命令格式** | `/compile` | `/graphify <path> [flags]` |
| **是否需要参数** | 无参数（固定处理 raw/） | 可指定路径，默认当前目录 |
| **所在位置** | `.claude/commands/compile.md` | `~/.claude/skills/graphify/` |

---

## graphify 完整命令清单

| 命令 | 用途 |
|------|------|
| `/graphify .` | 分析当前目录 |
| `/graphify ./raw` | 分析 raw 目录 |
| `/graphify ./raw --mode deep` | 深入提取（更多推断关系） |
| `/graphify ./raw --update` | 增量更新（只处理新/改文件） |
| `/graphify ./raw --cluster-only` | 重新聚类（不重新提取） |
| `/graphify ./raw --no-viz` | 跳过可视化，只生成报告+JSON |
| `/graphify ./raw --svg` | 额外导出 SVG |
| `/graphify ./raw --graphml` | 导出 GraphML（Gephi 用） |
| `/graphify ./raw --neo4j` | 生成 Neo4j 导入文件 |
| `/graphify ./raw --obsidian` | 生成 Obsidian Vault |
| `/graphify ./raw --watch` | 监控文件夹，自动更新 |
| `/graphify query "问题"` | BFS 查询（宽泛搜索） |
| `/graphify query "问题" --dfs` | DFS 查询（追踪路径） |
| `/graphify query "问题" --budget 1500` | 限制 token 预算 |
| `/graphify path "概念A" "概念B"` | 找两个概念的最短路径 |
| `/graphify explain "概念"` | 解释单个概念及其连接 |
| `/graphify add <url>` | 抓取 URL 并加入图谱 |
| `graphify hook install` | 安装 git commit 钩子 |

## 典型使用场景

### 场景 1：整理新收集的资料

```bash
# /compile - 生成摘要和概念笔记
/compile
```

```bash
# /graphify - 构建图谱
/graphify ./raw --update
```

**区别**：
- `/compile` → 生成 `wiki/summaries/`、`wiki/concepts/` 供阅读
- `/graphify` → 生成 `graph.html` 供可视化探索

---

### 场景 2：理解新代码库

```bash
# 只用 graphify，分析代码
/graphify ./src
```

```bash
# 分析结果
/graphify explain "AuthModule"
/graphify path "UserController" "Database"
```

**注意**：`/compile` 不适合分析纯代码，只处理 `raw/` 的文档资料。

---

### 场景 3：增量更新

```bash
# /compile - 手动判断哪些要编译
/compile   # 可能重复处理已有内容

# /graphify - 自动检测
/graphify . --update   # 只处理新/改的文件
```

---

### 场景 4：查询知识

```bash
# /compile - 读笔记后用自然语言问
# 需要先读 wiki/indexes/All-Concepts.md
# 然后让 AI 根据笔记内容回答

# /graphify - 直接查询图谱
/graphify query "Claude Code Skills 有哪些？"
/graphify query "Karpathy 和 graphify 什么关系？" --dfs
```

---

### 场景 5：持续监控

```bash
# /compile - 手动运行
# 每次想整理时运行

# /graphify - 自动监控
/graphify . --watch
# 代码改了就自动更新图谱
```

或者：
```bash
graphify hook install
# 每次 git commit 后自动更新
```

---

## 组合使用推荐

```bash
# 1. 收集资料后，先编译
/compile

# 2. 再构建图谱
/graphify ./raw --obsidian
# 生成 Obsidian Vault 到 graphify-out/obsidian/

# 3. 用 Obsidian 打开生成的 Vault 查看
```

## 快速选择指南

| 需求 | 推荐命令 |
|------|---------|
| 整理 raw/ 里的文章笔记 | `/compile` |

| 分析代码结构 | `/graphify ./src` |
| 增量更新 | `/graphify . --update` |
| 自然语言问答 | `/graphify query "问题"` |
| 找两个概念的联系 | `/graphify path A B` |
| 持续监听文件变化 | `/graphify . --watch` |
| 导出到 Gephi/Neo4j | `/graphify . --graphml` 或 `--neo4j` |
